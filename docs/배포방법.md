## 2. Docker 설치(아직 없다면)

Ubuntu 기준 예시입니다.

```bash
sudo apt-get update
sudo apt-get install -y ca-certificates curl
sudo install -m 0755 -d /etc/apt/keyrings
sudo curl -fsSL https://download.docker.com/linux/ubuntu/gpg -o /etc/apt/keyrings/docker.asc
sudo chmod a+r /etc/apt/keyrings/docker.asc
echo "deb [arch=$(dpkg --print-architecture) signed-by=/etc/apt/keyrings/docker.asc] https://download.docker.com/linux/ubuntu $(. /etc/os-release && echo "${VERSION_CODENAME:-$VERSION_ID}") stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
sudo apt-get update
sudo apt-get install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
sudo usermod -aG docker "$USER"
```

`usermod` 후에는 **로그아웃했다가 다시 SSH** 하거나, 당장만 `newgrp docker` 로 그룹을 적용합니다.

## 4. 기동

```bash
docker compose -f compose.prod.yml up -d --build
```

- 첫 실행은 이미지 빌드 때문에 시간이 꽤 걸릴 수 있습니다.
- 백엔드 `Dockerfile`의 `docker-entrypoint.sh`가 **`alembic upgrade head` 후 uvicorn**을 띄우므로, DB 마이그레이션은 컨테이너 기동 시 자동으로 돕니다.

## 참고(운영에서 자주 걸리는 것)

- **`.env.prod`의 `BINJARI_ENV=production`** 이면 리프레시 쿠키에 `Secure`가 붙습니다. **지금처럼 HTTP(IP)만** 쓰면 로그인/세션이 이상할 수 있어, 그때는 HTTPS를 쓰거나(도메인+Caddy) 임시로 환경을 조정하는 식으로 맞춥니다.
- DB 비밀번호에 `!` 등이 있으면 URL/환경 변수 이슈가 날 수 있어, 문제가 나면 비밀번호·인코딩을 의심하면 됩니다.

이 순서대로 하면 “Git 클론 → `.env.prod` → Compose 기동 → 브라우저로 접속”까지 이어집니다. 특정 단계에서 에러 메시지가 나오면 그 로그를 붙여 주면 다음 조치만 짚어 줄 수 있습니다.



원인: Docker Compose가 **`compose.prod.yml` 안의 `${DB_USER}`, `${DB_PASSWORD}`, `${DB_NAME}`를 채울 때** 서비스의 `env_file: .env.prod`는 **쓰지 않습니다.**

그 값들은 **같은 디렉터리의 `.env`** 또는 **`docker compose --env-file ...`** 로만 보간됩니다.  
서버에 `.env.prod`만 있고 `.env`가 없으면 Compose는 경고처럼 **빈 문자열**로 치환하고, Postgres 컨테이너에 사용자/DB 이름이 비어 가거나 잘못 들어가 **`binjari_db`가 비정상 → unhealthy**가 됩니다.

### 해결 방법 (하나만 하면 됨)

**방법 A — 권장 (한 줄)**  
프로젝트 루트에서:

```bash
docker compose --env-file .env.prod -f compose.prod.yml down
docker compose --env-file .env.prod -f compose.prod.yml up -d --build
```

**방법 B**  
Compose가 기본으로 읽는 이름으로 맞추기:

```bash
cp .env.prod .env   # 또는: ln -sf .env.prod .env
docker compose -f compose.prod.yml down
docker compose -f compose.prod.yml up -d --build
```

### 확인

```bash
docker compose --env-file .env.prod -f compose.prod.yml ps
docker compose --env-file .env.prod -f compose.prod.yml logs db --tail=30
```

`version` 경고는 Compose v2에서 무시해도 되고, 나중에 `compose.prod.yml` 맨 위 `version:` 줄을 지워도 됩니다.

요약: **`.env.prod`는 컨테이너 안으로는 들어가지만, YAML의 `${DB_*}` 치환에는 안 쓰이므로 `--env-file .env.prod` 또는 `.env` 복사가 필요합니다.**


docker compose -f compose.prod.yml restart caddy
